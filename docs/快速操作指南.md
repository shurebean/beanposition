## 🚀 快速开始（5分钟完成）

### 方式一：使用 initDatabase 云函数（推荐）

1. **上传 initDatabase 云函数**
   - 在微信开发者工具中，右键 `cloudfunctions/initDatabase`
   - 选择"上传并部署：云端安装依赖"

2. **调用云函数**
   - 在云开发控制台中，找到 `initDatabase` 云函数
   - 点击"调试"
   - 参数：
     ```json
     {}
     ```
   - 点击"调用"

3. **手动创建索引**（必须手动操作）
   - 在云开发控制台的"数据库" → "索引管理"中
   - 按照下面的索引配置表创建索引

4. **验证结果**
   - 检查 `themeParks` 集合：应有 18 条记录
   - 检查 `museums` 集合：应有 5 条记录

### 方式二：手动操作

#### 第 1 步：创建 6 个集合

| 序号 | 集合名称 |
|------|----------|
| 1 | users |
| 2 | checkins |
| 3 | museums |
| 4 | themeParks |
| 5 | userFavorites |
| 6 | dataSyncLog |

**操作方法：**
- 云开发控制台 → 数据库 → 添加集合
- 逐个创建上述 6 个集合

#### 第 2 步：创建索引（共 12 个）

**1. users 集合（1 个索引）**
| 索引名称 | 索引字段 | 类型 |
|---------|---------|------|
| openid | openid | 唯一 |

**2. checkins（3 个索引）**
| 索引名称 | 索引字段 | 类型 |
|---------|---------|------|
| userId | userId | 普通 |
| userId_visitDate | userId, visitDate | 复合 |
| timeline | timeline.year, timeline.month | 复合 |

**3. museums（3 个索引）**
| 索引名称 | 索引字段 | 类型 |
|---------|---------|------|
| province_city | province, city | 复合 |
| level | info.level | 普通 |
| type | info.type | 普通 |

**4. themeParks（2 个索引）**
| 索引名称 | 索引字段 | 类型 |
|---------|---------|------|
| brand | brand | 普通 |
| province_city | location.province, location.city | 复合 |

**5. userFavorites（2 个索引）**
| 索引名称 | 索引字段 | 类型 |
|---------|---------|------|
| userId | userId | 普通 |
| userId_placeType_placeId | userId, placeType, placeId | 唯一 |

**6. dataSyncLog（2 个索引）**
| 索引名称 | 索引字段 | 类型 |
|---------|---------|------|
| dataType | dataType | 普通 |
| syncTime | syncTime | 普通 |

#### 第 3 步：初始化数据

**同步游乐场数据：**
- 调用 `syncThemeParkData` 云函数
- 参数：`{}`

**同步博物馆数据：**
- 调用 `syncMuseumData` 云函数
- 参数：`{}`

## 📋 索引配置 JSON（复制到控制台）

```json
[
  {
    "collection": "users",
    "indexes": [
      {
        "name": "openid",
        "fields": ["openid"],
        "unique": true
      }
    ]
  },
  {
    "collection": "checkins",
    "indexes": [
      {
        "name": "userId",
        "fields": ["userId"]
      },
      {
        "name": "userId_visitDate",
        "fields": ["userId", "visitDate"]
      },
      {
        "name": "timeline",
        "fields": ["timeline.year", "timeline.month"]
      }
    ]
  },
  {
    "collection": "museums",
    "indexes": [
      {
        "name": "province_city",
        "fields": ["province", "city"]
      },
      {
        "name": "level",
        "fields": ["info.level"]
      },
      {
        "name": "type",
        "fields": ["info.type"]
      }
    ]
  },
  {
    "collection": "themeParks",
    "indexes": [
      {
        "name": "brand",
        "fields": ["brand"]
      },
      {

        "fields": ["location.province", "location.city"]
      }
```

## ⚠️ 重要提示

1. **索引必须手动创建**：云开发暂不支持通过代码自动创建索引
2. **集合名称区分大小写**：确保集合名称完全正确
3. **复合索引字段顺序**：注意复合索引中字段的顺序很重要
4. **唯一索引**：确保唯一索引字段的值不会重复
