## 🔄 常见问题

### 问题1：云函数上传失败

**症状：**
- 上传云函数时出现错误
- 提示"上传失败"或"部署失败"

**可能原因：**
1. 网络连接不稳定
2. 云开发环境ID不正确
3. 云开发环境未开通
4. package.json 配置错误
5. 依赖包安装失败

**解决方案：**
1. 检查网络连接
2. 检查云开发环境ID是否正确（bean-postion-0gwpv82hf2e100ed）
3. 检查云开发环境是否已开通
4. 检查 package.json 配置是否正确
5. 重新上传云函数
6. 查看控制台日志，确认错误信息

**排查步骤：**
```
1. 检查网络连接
2. 检查 project.config.json 中的 cloudfunctionRoot 配置
3. 检查查每个云函数的 package.json
4. 尝试上传单个云函数，定位问题
5. 查看微信开发者工具控制台日志
```

### 问题2：集合创建失败

**症状：**
- 创建集合时出现错误
- 提示"集合名称已存在"或"创建失败"

**可能原因：**
1. 集合名称已存在
2. 集合名称不符合规范
3. 云开发权限不足
4. 数据库未启用

**解决方案：**
1. 检查集合名称是否正确（区分大小写）
2. 检查集合名称是否符合规范（字母、数字、下划线）
3. 检查云开发权限
4. 检查数据库是否已启用
5. 重试创建

**集合名称规范：**
- 只能包含字母、数字、下划线
- 不能以数字开头
- 不能包含特殊字符
- 长度不超过 32 个字符
- 区分大小写

**集合名称建议：**
- ✅ users
- ✅ checkins
- ✅ museums
- ✅ themeParks
- ❌ userFavorites（可以，但建议使用 user-favorites）
- ❌ DataSyncLog（大小写错误，应为 dataSyncLog）

### 问题3：索引创建失败

**症状：**
- 创建索引时出现错误
- 提示"索引名称已存在"或"字段不存在"

**可能原因：**
1. 索引名称已存在
2. 索引字段不存在或名称错误
3. 索引类型配置错误
4. 唯一索引冲突

**解决方案：**
1. 检查索引名称是否唯一
2. 检查索引字段是否正确（区分大小写）
3. 检查字段路径是否正确（使用点号）
4. 检查唯一索引是否冲突
5. 重试创建

**字段路径示例：**
- ✅ info.level（正确）
- ❌ info-level（错误，应该使用点号）
- ✅ location.province, location.city（正确）
- ❌ location-province, location-city（错误）

**症状：**
- 调用`initDatabase`云函数时出现错误
- 提示"云函数不存在"或"执行失败"

**可能原因：**

2. 云函数名称错误
3. 参数格式不正确
4. 云函数内部逻辑错误
5. 数据库连接失败

**解决方案：**
1. 检查云函数是否已部署
2. 检查云函数名称是否正确
3. 检查参数格式是否正确（JSON 格式）
4. 查看云函数日志
5. 检查数据库集合是否已创建
6. 重新调用云函数

**排查步骤：**
