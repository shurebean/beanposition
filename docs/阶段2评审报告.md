## 📋 数据库初始化方案

### 方案对比

#### 方案 A：完全手动操作（适合初学者）

**优点：**
- 理解每一步操作
- 出错容易定位
- 不依赖复杂脚本

**缺点：**
- 操作步骤多（30+ 步）
- 容易遗漏
- 耗时较长

**操作时间：** 约 15-20 分钟

#### 方案 B：半自动化（推荐）⭐

**优点：**
- 使用 initDatabase 云函数自动创建集合
- 自动初始化数据
- 只需手动创建索引（12 个）
- 出错有详细日志

**缺点：**
- 索引仍需手动创建（微信云开发限制）

**操作时间：** 约 5-10 分钟

#### 方案 C：完全自动化（未来优化）

**优点：**
- 一键完成所有操作
- 无需手动干预

**缺点：**
- 微信云开发暂不支持通过 API 创建索引
- 需要等待平台支持

**操作时间：** 约 2-3 分钟

**推荐方案：方案 B（半自动化）**

**功能：**
- ✅ 自动检查和创建 6 个数据库集合

**参数：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| forceRefresh | Boolean | false | 是否强制刷新数据 |
| skipData | Boolean | false | 是否跳过数据初始化 |

**调用示例：**

```json
{
  "forceRefresh": true
}
```

**返回示例：**

```json
{
  "success": true,

  "collections": {
    "created": [],

  "data": {

    "museums": { "count": 5 }
  }
}
```
