## 方法一：微信开发者工具操作（推荐）

### 步骤 1：打开云开发控制台

1. 在微信开发者工具中，点击顶部菜单栏的"云开发"按钮
2. 进入云开发控制台

### 步骤 2：创建数据库集合

#### 创建 users 集合

1. 在云开发控制台左侧菜单中，点击"数据库"
2. 点击"添加集合"
3. 集合名称：`users`
4. 点击"确定"

**创建索引：**
1. 点击 `users` 集合进入详情页
2. 点击"索引管理" → "添加索引"
3. 索引名称：`openid`
4. 索引字段：`openid`
5. 索引类型：唯一索引
6. 点击"确定"

#### 创建 checkins 集合

1. 返回数据库列表页
2. 点击"添加集合"
3. 集合名称：`checkins`
4. 点击"确定"

**创建索引：**

| 索引名称 | 索引字段 | 索引类型 |
|---------|---------|---------|
| userId | userId | 普通索引 |
| userId_visitDate | userId, visitDate | 复合索引 |
| timeline | timeline.year, timeline.month | 复合索引 |

1. 点击 `checkins` 集合进入详情页
2. 点击"索引管理" → "添加索引"
3. 按上表依次创建 3 个索引
4. 复合索引的设置方法：点击"添加字段"，输入多个字段名

#### 创建 museums 集合

1. 返回数据库列表页
2. 点击"添加集合"
3. 集合名称：`museums`
4. 点击"确定"

**创建索引：**

| 索引名称 | 索引字段 | 索引类型 |
|---------|---------|---------|
| province_city | province, city | 复合索引 |
| level | info.level | 普通索引 |
| type | info.type | 普通索引 |

#### 创建 themeParks 集合

1. 返回数据库列表页
2. 点击"添加集合"
3. 集合名称：`themeParks`
4. 点击"确定"

**创建索引：**

| 索引名称 | 索引字段 | 索引类型 |
|---------|---------|---------|
| brand | brand | 普通索引 |
| province_city | location.province, location.city | 复合索引 |

#### 创建 userFavorites 集合

1. 返回数据库列表页
2. 点击"添加集合"
3. 集合名称：`userFavorites`
4. 点击"确定"

**创建索引：**

| 索引名称 | 索引字段 | 索引类型 |
|---------|---------|---------|
| userId | userId | 普通索引 |
| userId_placeType_placeId | userId, placeType, placeId | 唯一索引（复合）|

#### 创建 dataSyncLog 集合

1. 返回数据库列表页
2. 点击"添加集合"
3. 集合名称：`dataSyncLog`
4. 点击"确定"

**创建索引：：**

| 索引名称 | 索引字段 | 索引类型 |
|---------|---------|---------|
| dataType | dataType | 普通索引 |
| syncTime | syncTime | 普通索引 |

## 常见问题排查

### 问题 1：云函数调用失败

**错误信息：**
```
Error: cloud function is not deployed
```

**解决方案：**
1. 检查云函数是否已上传和部署
2. 在微信开发者工具中，右键点击云函数文件夹
3. 选择"上传并部署：云端安装依赖"

### 问题 2：数据库集合不存在

**错误信息：**
```
Error: collection not found
```

**解决方案：**
1. 确认是否已创建所有 6 个集合
2. 检查集合名称是否正确（区分大小写）
3. 参考本文档的"创建数据库集合"部分重新创建

### 问题 3：索引创建失败

**错误信息：**
```
Error: index already exists
```

**解决方案：**
1. 此错误通常可以忽略
2. 表示索引已经存在
3. 可以在"索引管理"中查看已存在的索引

### 问题 4：数据写入失败

**错误信息：**
```
Error: permission denied
```

**解决方案：**
1. 检查数据库权限配置
2. 确保云函数有写入权限
3. 在云开发控制台的"数据库" → "权限设置"中检查

### 问题 5：数据重复

**错误信息：**
```
Error: duplicate key error
```

**解决方案：**
1. 数据已经存在
2. 可以使用 `forceRefresh` 参数强制刷新：
   ```json
   {
     "forceRefresh": true
   }
   ```
3. 这会删除旧数据并重新写入

### 问题 6：游乐场数据文件不存在

**错误信息：**
```
Error: Cannot find module './theme-parks.json'
```

**解决方案：**
1. 检查 `cloudfunctions/syncThemeParkData/theme-parks.json` 文件是否存在
2. 如果不存在，从 `data/theme-parks.json` 复制过来

**复制命令：**
```bash
cp /home/beanposition/data/theme-parks.json /home/beanposition/cloudfunctions/syncThemeParkData/theme-parks.json
```

## 自动化脚本说明

### 脚本位置

- 完整初始化脚本：`scripts/init-database.js`
- 游乐场数据同步云函数：`cloudfunctions/syncThemeParkData/index.js`
- 博物馆数据同步云函数：`cloudfunctions/syncMuseumData/index.js`

### 脚本功能

#### syncThemeParkData 云函数

- **功能**：同步游乐场数据到数据库
- **数据来源**：`cloudfunctions/syncThemeParkData/theme-parks.json`
- **数据量**：18 条
- **特性**：
  - 支持增量同步（检查数据是否已存在）
  - 支持强制刷新（`forceRefresh: true`）
  - 自动记录同步日志

**调用参数：**

```json
{
  "forceRefresh": true
}
```

#### syncMuseumData 云函数

- **功能**：同步博物馆数据到数据库
- **数据来源**：GitHub API（当前使用示例数据）
- **数据量**：5 条（示例数据）
- **特性**：
  - 支持增量同步（检查数据是否已存在）
  - 支持强制刷新（`forceRefresh: true`）
  - 自动记录同步日志
  - 支持数据验证和去重

**调用参数：**

```json
{
  "forceRefresh": true
}
```

### 注意事项

1. **数据文件位置**：
   - `syncThemeParkData` 云函数依赖 `theme-parks.json` 文件
   - 该文件必须放在 `cloudfunctions/syncThemeParkData/` 目录下
   - 不要只依赖 `data/` 目录下的文件

2. **云函数权限**：
   - 确保云函数有数据库读写权限
   - 在云开发控制台的"云函数" → "权限设置"中配置

3. **数据库权限**：
   - 确保数据库集合已创建
   - 确保索引已创建
   - 检查权限规则是否正确

4. **网络请求**：
   - 云函数中发起的网络请求必须通过 HTTPS
   - 注意云函数的网络访问限制
   - 博物馆数据同步需要配置 GitHub API 访问

### 第一次部署（从零开始）

```

   ↓
```
