# 配置信息安全保存指南

## 📁 配置文件保存位置

所有敏感配置信息将保存在项目本地的以下位置：

```

```

⚠️ **重要提醒：** `.env` 文件已加入 `.gitignore`，不会被提交到 Git 或 GitHub。

```

```

**权限设置：**
- 只有 root 用户可读写
- 文件不会被提交到 Git
- 文件不会被上传到 GitHub

**验证方式：**

## ❓ 常见问题

### Q1: 配置信息会被泄露吗？

**A: 不会。** 
- 配置文件保存在服务器本地
- 文件权限设置为 600（仅 root 可读写）
- 文件已加入 `.gitignore`，不会提交到 Git
- Git 仓库已推送到 GitHub，但不包含 `.env`

### Q2: 如何验证配置信息安全？

**A: 运行以下命令验证：**

```bash
# 1. 检查文件权限
ls -l /home/beanposition/config/.env

# 2. 检查是否被 Git 忽略
cd /home/beanposition
git check-ignore -v config/.env

# 3. 检查 Git 状态（不应显示 .env）
git status

# 4. 检查 Git 提交历史（不应包含密钥）
git log --all --source --full-history -S "密钥" -- "*.env"
```

### Q3: 如果需要修改配置信息怎么办？

**A: 我会：**
1. 使用 `vi` 或 `echo` 命令编辑 `.env` 文件
2. 修改后验证文件权限
3. 确认不会被 Git 追踪
4. 重新加载配置

### Q4: 可以把配置信息写在环境变量里吗？

**A: 可以，但建议使用文件。**

**环境变量方式：**
```bash
export WECHAT_APP_ID=wx123456789abcdef
export WECHAT_APP_SECRET=32位字符串
```

**文件方式（推荐）：**
- 配置集中管理
- 不受 Shell 会话影响
- 更易于版本控制（模板）

### Q5: 配置信息会备份吗？

**A: 不会。** 
`.env` 文件不会被 Git 提交，也不会被 GitHub 备份。

**如果你需要备份配置信息：**
1. 保存到本地安全位置
2. 使用密码管理器（如 1Password）
3. 使用云密钥管理服务（如腾讯云 KMS）
